```bash
# 1. Подготовьте окружение (10 мин)
cp .env.example .env

docker-compose down -v
docker-compose up -d

# 3. Проверьте все (10 мин)
python3 << 'EOF'
import subprocess
import requests
import time

services = [
    ("Gateway", 8005),
    ("User Service", 8000),
    ("Notification Service", 8001),
    ("Event Service", 8002),
    ("Admin Service", 8003),
    ("Maps Service", 8004),
    ("Media Service", 8006)
]

print("=" * 50)
print("STARTUP VERIFICATION")
print("=" * 50)

for name, port in services:
    try:
        resp = requests.get(f"http://localhost:{port}/health", timeout=2)
        if resp.status_code == 200:
            print(f"✓ {name:20} [OK] Port {port}")
        else:
            print(f"✗ {name:20} [HTTP {resp.status_code}] Port {port}")
    except Exception as e:
        print(f"✗ {name:20} [ERROR] {str(e)[:30]}")

print("=" * 50)
EOF

# 4. Проверьте флаги (10 min)
python3 ctf_validator.py --list-flags | head -20

# 5. Создайте тестовых пользователей (15 min)
python3 ctf_poc_exploits.py --register
# Создайте ещё несколько...

# 6. Объявляйте готовность! (5 min)
# Разошлите ссылки на документацию участникам
```

## 🔧 КАСТОМИЗАЦИЯ

### Как изменить количество уровней?

Отредактируйте `ctf_validator.py`:
```python
FLAGS_DATABASE = [
    # Удалите/добавьте Flag() объекты
    Flag(level=1, ...),
    Flag(level=2, ...),
    # Добавьте новые:
    Flag(level=21, content="new_flag", ...),
]
```

### Как добавить новый уровень?

1. Добавьте в CTF_SCENARIO.md новый раздел со всеми деталями
2. Добавьте уязвимость в код (используя CTF_IMPLEMENTATION_GUIDE.md как шаблон)
3. Добавьте Flag() объект в ctf_validator.py
4. Добавьте функцию-эксплуатацию в ctf_poc_exploits.py
5. Протестируйте что it works: `curl ... | grep FLAG`

### Как изменить баллы?

В ctf_validator.py:
```python
Flag(
    level=1,
    content="...",
    points=10,  # ← Измените это
    ...
)
```

## 📈 МЕТРИКИ И ОТСЛЕЖИВАНИЕ

### Что отслеживать?

```bash
# Общие метрики
python3 ctf_validator.py --export progress.json
cat progress.json | jq '.by_category'

# Specific флаг
curl -s http://localhost:8000/logs | grep "FLAG{" | wc -l

# Performance
time python3 ctf_poc_exploits.py --full
```

### Что это говорит нам?

- **Если 80%+ решили Flag 1-3:** Участники поняли базы
- **Если <20% решили Flag 10+:** Нужны лучшие подсказки
- **Если средний time > 30 мин per flag:** Возможно, слишком сложно
- **Если никто не решил Flag 20:** Нужно проверить что уязвимость работает

## 🆘 TROUBLESHOOTING

### "Сервисы не запускаются"
```bash
docker-compose logs --tail=100
# Ищите ошибки в БД, миграциях, зависимостях
docker-compose down -v && docker-compose up --build
```

### "Флаги не работают"
```bash
python3 ctf_validator.py --validate "FLAG{test}"
# Проверьте точный формат и content
```

### "Участники не могут решить уровень"
```bash
python3 ctf_poc_exploits.py --stage 6  # Проверьте что уязвимость работает
# Раздайте подсказку из CTF_SCENARIO.md
```

### "Performance слишком медленный"
```bash
# Может быть Docker на медленной машине
docker stats  # Посмотрите load
# Увеличьте ресурсы в docker-compose.yml
```

## 📞 ПОДДЕРЖКА
 
Если что-то не работает:

1. **Проверьте документацию** - почти все вопросы там
2. **Запустите sanity check:**
   ```bash
   cd shoulder-to-shoulder
   bash ctf_sanity_check.sh 2>/dev/null || python3 ctf_poc_exploits.py --full
   ```
3. **Посмотрите логи:**
   ```bash
   docker-compose logs -f | grep -i error
   ```
4. **Заново создайте окружение:**
   ```bash
   docker-compose down -v
   docker volume prune
   docker-compose up --build -d
   ```

## 🎓 ПОСЛЕ CTF

### Разбор уязвимостей (2-3 часа)

```markdown
1. **Участники представляют решения** (30 мин)
   - Какой флаг решили?
   - Какой был самый сложный?
   
2. **Организаторы показывают правильные решения** (60 мин)
   - Демонстрация POC
   - Объяснение каждой уязвимости
   - Best practices для защиты
   
3. **Discussion вызовов** (30 мин)
   - Что было неясно?
   - Какие инструменты помогли?
   - Как это применено в реальной жизни?
```

### Write-up документация

```bash
# Попросите у участников написать write-up для каждого флага

# Пример структуры write-up:
- Название флага
- Что нужно было сделать
- Основные шаги решения
- Инструменты/команды которые использовались
- Скриншоты (опционально)
- Что изучили из этого
```
