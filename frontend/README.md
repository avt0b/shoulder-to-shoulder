# 🏋️ Плечом к плечу — Frontend

Мобильное веб-приложение на **Vue 3 + Leaflet** для поиска спортивных мест и групповых тренировок в городе Орёл.

## 📁 Структура

```
frontend/
├── app/                    # Основное приложение (Vue 3 + Vite)
│   ├── src/
│   │   ├── components/
│   │   │   ├── MainPage.vue    # Главная страница с мини-картой
│   │   │   └── MapLight.vue    # Полноэкранная карта с маркерами
│   │   ├── App.vue             # Роутинг MainPage ↔ MapLight
│   │   ├── config.js           # Конфигурация API
│   │   ├── style.css           # Глобальные стили + Material Design
│   │   └── main.js
│   ├── .env.example            # Шаблон переменных окружения
│   ├── vite.config.js
│   └── package.json
├── API.md                  # Полная документация по API
├── BACKEND.md              # Гайд для бэкенд-разработчика
└── .gitignore
```

## 🚀 Быстрый старт

```bash
cd app
npm install
npm run dev
```

Приложение запустится на `http://localhost:5173`

## 🔌 API конфигурация

Скопируйте `.env.example` в `.env` и настройте:

```bash
cp .env.example .env
```

| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `VITE_API_BASE_URL` | URL вашего бэкенда | `http://localhost:3000/api/v1` |
| `VITE_OSRM_BASE_URL` | URL OSRM сервера | `https://router.project-osrm.org` |
| `VITE_APP_ENV` | Окружение | `development` |
| `VITE_DEBUG` | Логи в консоли | `true` |

## 📡 API Endpoints

Бэкенд должен предоставить REST API. Подробнее:
- **[API.md](API.md)** — полная документация с примерами запросов/ответов
- **[BACKEND.md](BACKEND.md)** — гайд для бэкенд-разработчика

### Краткая сводка

| # | Метод | Эндпоинт | Что делает |
|---|-------|----------|------------|
| 1 | `GET` | `/places` | Все места для карты |
| 2 | `GET` | `/places/{id}` | Одно место |
| 3 | `GET` | `/route` | Маршрут (прокси OSRM) |
| 4 | `GET` | `/places/nearby` | Ближайшие места |
| 5 | `GET` | `/meetups/my` | Мои мероприятия |
| 6 | `POST` | `/meetups/{id}/join` | Записаться |
| 7 | `DELETE` | `/meetups/{id}/leave` | Отписаться |

> **💡 Минимальный старт:** достаточно реализовать эндпоинты **1** и **5** для начала.

## 🗺 Координаты

Центр города Орёл: `[52.9651, 36.0785]`

**Важно:** OSRM использует формат `[долгота, широта]` → `[lng, lat]`!

## 📱 Для бэкендера

Если вы бэкенд-разработчик, начните с изучения файлов:
1. **`API.md`** — что именно ожидать от API
2. **`BACKEND.md`** — как фронтенд использует API
3. **`app/src/config.js`** — где настраивается URL бэкенда

Сейчас фронтенд работает с **mock-данными**. Для подключения реального API нужно:
1. Поднять бэкенд на `http://localhost:3000`
2. Реализовать минимум `GET /api/v1/places` и `GET /api/v1/meetups/my`
3. Фронтенд автоматически переключится на API (fallback на моки при ошибке)
